Recombination Sensitivity

The purpose of this sensitivity analysis is to understand how the results of the differential evolution algorithm respond to different recombination constant. A total of 6 different recombination constants (0.4, 0.5, 0.6, 0.7, 0.8, 0.9) were tested against every iteration of the Distributed Representation initial condition sets (All, No Min/Mean Thickness, Volume, & Poiseuille, No Min/Mean Thickness, & Volume, and Reduced).

This project uses the SciPy function: optimize.differential_evolution. The seed was kept constant at 1 to ensure experiment reproducibility; each structural attribute weight was bounded between 0 and 10; the mutation constant was the tuple (0.5,1) which employs dithering (dithering randomly changes the mutation constant on a generation by generation basis); the strategy was "best1bin’" meaning the best current solution vector and 2 random population vectors are used for creating a mutated vector; the absolute tolerance was set to 0 and the relative tolerance was set to 0.01; polish was set to true meaning scipy.optimize.minimize is used to polish the best population member at the end, which can improve the minimization slightly; and the initial population was set by one of the 4 options described above. With those function input parameters set, the recombination constant was changed for each experiment to compare the accuracy, runtime, and weights assigned to each structural parameter, as shown in Tables 1 through 8.

Tables 1, 3, 5, & 7 show the weight assigned to each structural attribute for a given recombination constant; the same information is shown graphically in Figures 1, 2, 3, & 4. Immediately, one will notice that every CR produces the same weighting scheme, highlighting only the Aspect Ratio attribute; this was the same for every single initial condition set . Thus, regardless of the initial condition and/or the recombination constant, when considering the 100%, 77%, & 71% saturated samples, there will be no difference to the resulting weighting scheme. Note, this was not the case when considering the 83%, 77%, & 71% saturated samples in combination. This is likely due to the interesting structure of the percolating pathway in the 83% saturated sample compared to the 100% saturated sample.

Tables 2, 4, 6, & 8 show the respective accuracy for each level of saturation, the mean accuracy, and the program runtime (time to convergence) in seconds.

All of the recombination constants performed identically in terms of accuracy and as CR decreased, the runtime generally increased. However, all of the resulting runtimes are acceptable for upscaling to larger training sets.
