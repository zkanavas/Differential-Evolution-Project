Initial Condition Sensitivity

This project is searching for a quantifiable relationship between pore network structure and velocity distributions. With a known percolating pathway, the project compares the shortest path, using Dijkstra’s algorithm (weighted by various structural attributes), to determine the level of overlap or accuracy . If a path is highly accurate then the weighting scheme used to produce it is said to be influential on the development of preferential flow pathways. The structural attributes explored include:

To search through what (currently linear) combinations of these structural attributes produces an accurate shortest path, this project employed differential evolution – a genetic algorithm. This algorithm will continuously change the weight multiplied against the structural attribute value to make it more or less impactful to the overall weighting scheme, comparing each population member to find the “fittest” or “most accurate” weighting scheme.

The purpose of this sensitivity analysis is to understand how the results of the differential evolution algorithm respond to different provided initial conditions. A total of 4 different initial conditions were provided to the program:

LatinhypercubeDistributed Representation(DR) – AllDR – Not included: Min/Mean Thickness & Volume AttributesDR – Not included: Min/Mean Thickness, Volume, & Poiseuille AttributesDR – Reduced

Each set of initial conditions is explained further in subsequent sections, however it should be noted that “latinhypercube” is a population initialization arguments whereas the rest of the initial conditions are user-defined.

This project uses the SciPy function: optimize.differential_evolution. Population initialization using this function requires either a function argument (latinhypercube or random) or an array specifying the initial population. If providing an array, there must be at least 5 vectors. This is because the some of the available mutation strategies require 5 vectors (the best current solution + 4 random population vectors) to create a mutated vector.

The seed was kept constant at 1 to ensure experiment reproducibility; each structural attribute weight was bounded between 0 and 10; the mutation constant was the tuple (0.5,1) which employs dithering (dithering randomly changes the mutation constant on a generation by generation basis); the recombination constant was 0.7; the strategy was "best1bin’" meaning the best current solution vector and 2 random population vectors are used for creating a mutated vector; the absolute tolerance was set to 0 and the relative tolerance was set to 0.01; and polish was set to true meaning scipy.optimize.minimize is used to polish the best population member at the end, which can improve the minimization slightly. With those function input parameters set, the initial population was set by one of the nine options for each experiment to compare the accuracy, runtime, and weights assigned to each structural parameter, as shown in Tables 1 & 2.
